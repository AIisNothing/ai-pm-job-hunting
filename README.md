# AI时代下的产品经理怎么找工作

> 一个面向求职者的 AI 求职市场分析方法案例。用 AI 从公开招聘信息中提炼市场信号，辅助更理性的求职决策。

## 🎯 为什么做这件事

传统的找工作方式本质上是随机的——自己在招聘平台上翻岗位、凭感觉投简历，或者等猎头推荐。你很难回答"现在市场到底什么方向值钱""我的经验适合往哪走""高薪岗位到底多要求了什么"这些关键问题，因为你看到的永远只是局部信息。

AI 时代给了我们一个新的可能：用数据代替直觉。

AI 擅长处理大规模数据、发现人眼难以察觉的趋势、做多维度交叉分析。当我们把一批公开招聘信息样本中的薪资、方向、技能要求和公司类型做聚合分析，很多原本模糊的判断就变得清晰了——哪些方向高薪岗位密集、哪些"热门方向"其实薪资平平、高薪 JD 和普通 JD 的能力差异到底在哪里。

这个项目就是一次实践：**AI 不是替你找工作，而是帮你更理性地做求职决策。**

## 💡 这个项目能帮你做什么

- **理解市场需求**：哪些方向岗位多、哪些方向高薪密集、哪些方向看起来热但薪资平平
- **判断方向价值**：不只看中位数，还看 P75/P90、高薪占比、样本可信度
- **识别能力要求**：高薪 JD 和普通 JD 的差异不在工具技能，而在业务闭环、指标驱动、策略设计等能力因子
- **拆解 JD 信号**：区分"方向标签"和"能力信号"，避免把标签当技能
- **转化为决策**：把市场数据转化为"投什么方向、补什么能力、怎么包装经历"的具体行动
- **提供可复用框架**：一套完整的分析 Prompt、报告生成代码和审计规范，你可以用自己的数据跑

## 🔑 示例报告观察（基于特定时间与筛选条件下的公开招聘信息样本）

> 样本范围：北京 · 3-10年经验 · 本科+ · 月薪30K+ · 全职
> 
> ⚠️ 以下仅为方法示例产出，反映特定样本的聚合趋势，不代表完整市场，不构成求职建议。

**1. 市场基准**

薪资中位数 37.5K/月，P75 为 45.0K，P90 达 55.0K。大部分岗位集中在 30-50K 区间，突破 50K 需要在方向选择或职级上有突破。

**2. 方向选择**

- **高薪岗位密集**：策略产品（高薪占比 36.4%）、AI/大模型产品（P75 溢价 +5.6%，高薪占比 36.3%）
- **高薪机会多但中位数无溢价**：电商/交易、金融产品、增长产品、商业化/广告——需精准筛选 JD
- **天花板高但需要实力**：用户体验/C端（P90=58.5K）
- **保底方向**：B端/SaaS/企服、数据产品——岗位多但薪资中等
- **小样本观察**：平台产品、内容/社区——样本太少，不建议据此决策

**3. 高薪能力因子**

高薪 JD 和普通 JD 的核心差异不在于工具技能，而在于：
- 业务闭环（对结果负责）
- 指标驱动（用数据判断价值）
- 策略设计（设计策略并验证效果）
- AI 应用落地
- 复杂系统抽象
- Owner 能力

**4. 关键提醒**

- 标签≠能力：高频标签只能说明"高薪岗位中更常见"，不能证明因果
- 中位数≠全貌：多数方向中位数持平，差异主要体现在 P75/P90 和高薪占比
- 小样本需谨慎：<50 个岗位的方向结论不够稳定

## 方法论

### 分析流程

```
公开招聘信息样本 → 薪资标准化 → 方向重新聚类 → 极端值剔除 → 多维交叉分析 → 脱敏报告生成
```

### 分析维度

- **方向分析**：17 个方向重新聚类（不依赖平台原始分类）
- **薪资分层**：中位数 / P75 / P90 / 高薪占比（P75+）
- **能力解构**：高薪 JD vs 普通 JD 的能力因子差异
- **技能分类**：方向标签 / 能力信号 / 小样本趋势词 / 无区分度标签
- **交叉验证**：经验×方向、行业×薪资、公司类型×高薪占比

### 审计规范

每次生成报告后执行：
- 数据溯源审计（独立复算核心指标）
- 数字一致性审计（同一指标跨章节一致）
- 口径一致性审计（不混用不同统计口径）
- 结论支撑审计（每个结论有数据依据）
- 小样本警告（<50 标注谨慎）

## 技术栈

- **数据存储**：SQLite + SQLAlchemy
- **数据分析**：numpy
- **报告生成**：Python 生成自包含 HTML（内嵌 ECharts）
- **图片导出**：Playwright 全页截图（2x Retina）


> ⚠️ **数据源说明**：运行分析代码需要你自己准备数据。请使用你有权处理的数据源、手动整理的公开招聘信息样本，或模拟数据。本项目不提供也不鼓励抓取任何特定平台数据。
